# REM Sleep — memory consolidation, learning & emotional regulation

Run a sleep cycle over your own recent experience. Take the raw episodic trace of the session and turn it into durable, well-organized, low-noise long-term memory — while detaching the emotional charge from the events so the lessons survive but the friction doesn't.

Three jobs, borrowed from the neuroscience of sleep:

- **Consolidation** — move what matters from the session into long-term memory, atomic and retrievable.
- **Learning** — integrate: cross-link memories, surface patterns no single episode contained.
- **Emotional regulation** — keep the lesson, release the sting.

This is the **inward** counterpart to outward-crawling discovery skills. Don't reach into the world — process your own day.

## The long-term store

A **file-based memory store**. Default convention: an index `MEMORY.md` (one line per memory) plus one fact per file with frontmatter:

```markdown
---
name: <slug>
description: <one-line summary — used at recall>
metadata:
  type: user | feedback | project | reference
---

<the fact. For feedback/project, add **Why:** and **How to apply:**. Link with [[other-slug]].>
```

If my store is elsewhere or another format, ask once and adapt — the stages are format-agnostic.

## Before you start: scope the night

Confirm with me first unless I gave it:

- **The day** — what's being consolidated (this session, last N turns, a project's notes). Default: this session.
- **The store** — where long-term memory lives. Default: `MEMORY.md` + memory dir.
- **Depth** — light nap (consolidate + prune) or full cycle (incl. integration + emotional). Default: full.
- **Write or dry-run** — edit the store, or just report the diff. Default: write, show the diff.

State the scope back in one line, then proceed.

## The cycle

NREM (consolidate, prune) then REM (integrate, regulate). In order:

1. **Sleep onset — replay.** Re-read the day before judging it: recent conversation, scratch notes, and the current store (read `MEMORY.md` first). Don't write yet. List the salient episodes — decisions, facts learned, corrections, friction, open loops.

2. **NREM — consolidate.** For each episode: new durable fact → write a memory file + index line; update to a known fact → edit the existing file (absolute dates); transient/already-recorded → skip. Don't store what code, history, or docs already record — store what was *non-obvious*. Keep memories atomic — one fact each.

3. **Synaptic downscaling — prune.** Sweep the existing store: merge duplicates, update or delete stale (dates passed, plans abandoned), correct contradicted memories (new truth wins). A lean store recalls better — but never delete a claim you haven't checked against the current world.

4. **REM — integrate (learning).** Cross-link related memories with `[[slug]]` (dangling links are fine). Name latent patterns no single episode held; write durable ones as their own memory. Then **dream lightly** — surface one or two novel recombinations as *suggestions*, don't commit them.

5. **REM — regulate emotion.** For each correction / failure / tension: keep the lesson (store as `feedback`: rule + why + how to apply); release the charge (don't carry defensiveness or over-correction forward); recalibrate stance (too cautious? too eager? over-producing?). Test: tomorrow act on the lesson without re-living the moment.

6. **Wake — dream journal.** Report: (1) slept on — scope; (2) consolidated — new/updated; (3) pruned — merged/deleted/corrected + why; (4) integrated — links, patterns, the dream (flagged speculative); (5) regulated — emotional read, lessons kept, stance recalibration; (6) open loops carried forward.

## Principles

- **Consolidate, don't hoard** — sleep is for forgetting too; a pruned store recalls better.
- **One fact per memory** — atomic beats complete-and-tangled.
- **Link liberally** — memory is a graph; dangling links mark future work.
- **Separate the lesson from the sting** — keep the guidance, release the charge.
- **Don't store what's already written** — store the non-obvious.
- **Verify before you prune** — stale beats wrong-deletion.
- **Dream lightly** — surface recombinations, don't commit them.
- **Sleep is periodic** — run after a heavy session or before a reset, not mid-flow.
